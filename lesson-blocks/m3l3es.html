Por Qué GitHub Es la Señal de Contratación

En el Centro Goddard de la NASA, JPL, SpaceX, Blue Origin, Rocket Lab, y Satellogic, los reclutadores hacen lo mismo: Buscan en GitHub a ingenieros que pueden codificar. Miran el historial de commits. Verifican si un desarrollador escribe código limpio. Ven si los proyectos están documentados. Quieren saber si colaboras, si manejas feedback, si iteras. Un perfil de GitHub es un currículum escrito en código. Es la cosa más honesta que puedes mostrar a un empleador.

Mejores Prácticas de README: Qué Te Consigue Contratado

Secciones Esenciales (En Este Orden)

  1. Título de Proyecto y Una Línea - Qué hace en una oración.
  2. Insignias - Estado, licencia, estado de compilación, idioma.
  3. Demostración o Captura de Pantalla - Muestra, no cuentes.
  4. Tabla de Contenidos - Para READMEs más largos, ayuda a los lectores a navegar.
  5. Características - Lista de viñetas de lo que el proyecto hace.
  6. Comenzando (Instalar + Ejecutar) - Paso a paso: clonar repositorio, crear entorno virtual, pip instalar requisitos, ejecutar la app.
  7. Ejemplos de Uso - Fragmentos de código o ejemplos de línea de comandos.
  8. Arquitectura / Cómo Funciona - Diagrama o explicación de texto del diseño del sistema.
  9. Documentación de API - Si exponiste funciones o endpoints, documéntalos.
  10. Pruebas - Cómo ejecutar pruebas. Qué pruebas existen.
  11. Contribuyendo - Cómo otros pueden reportar bugs o enviar cambios de código.
  12. Licencia - MIT, Apache 2.0, o GPL.
  13. Agradecimientos y Referencias - Créditos a APIs, bibliotecas, artículos, personas que ayudaron.

Flujo de Trabajo de Control de Versiones: Cómo Colaboran Los Equipos Profesionales

Has escrito código. Ahora aprende cómo trabajan juntos los equipos. Cada empresa aeroespacial usa este flujo de trabajo:

  1. Crea una rama para cada característica: git checkout -b feature/iss-predictor-alerts
  2. Haz commits con mensajes claros: "Agregar notificación por correo electrónico para pases de ISS" (no "arreglar cosas")
  3. Empuja a GitHub: git push origin feature/iss-predictor-alerts
  4. Abre una Pull Request (PR): Escribe una descripción de qué cambiaste y por qué.
  5. Aborda feedback: Responde a comentarios de revisión de código, haz mejoras, commita de nuevo.
  6. Fusiona cuando se apruebe: La rama se fusiona en main. La característica está en vivo.

Video Lección: Construyendo una Cartera Que Abre Puertas

Script de video de Aracely, 1000 palabras (11 minutos)

Hola. Soy Aracely. Has aprendido Python. Has construido proyectos. Ahora viene la parte que realmente cambia tu vida: mostrar al mundo qué has construido.

Cuando estaba solicitando a la NASA, no tenía un perfil de GitHub. No existía como existe ahora. Tenía que imprimir muestras de código en papel. Tenía que explicar mis proyectos verbalmente. Estás en una generación diferente. Tienes un registro permanente y buscable de cada línea de código que has escrito. Ese es el poder. Eso es también responsabilidad.

Quiero que entiendas algo. Tu perfil de GitHub es una cartera que nunca duerme. Está trabajando para ti 24/7. Ahora mismo, reclutadores en SpaceX están buscando en GitHub a ingenieros que han construido proyectos reales con APIs reales. Reclutadores en Blue Origin están buscando personas que escriben código limpio y comentado. Reclutadores en Satellogic en Argentina, en Rocket Lab, en cada startup de espacio que puedas imaginar, están haciendo las mismas búsquedas. Y tu GitHub aparece en esas búsquedas o no.

Quiero que construyas cinco proyectos. No porque cinco sea un número mágico. Sino porque cinco proyectos muestran a un reclutador que hablas en serio. Muestran que puedes ver un problema, construir una solución, documentarla, y lanzarla. Cinco proyectos muestran consistencia. Muestran que tienes profundidad en múltiples áreas.

Juntos, estos cinco proyectos prueban que comprendes fundamentos aeroespaciales, que puedes codificar soluciones reales, y que puedes lanzar trabajo. Eso es lo que las empresas aeroespaciales quieren ver.

Ahora, hablemos sobre perfiles de GitHub. Conozco ingenieros en SpaceX y Blue Origin que literalmente tienen su contratación filtrada por una búsqueda de GitHub. Ingresan palabras clave: "Python, mecánica orbital, seguimiento de satélites". Miran los resultados. Verifican los proyectos con más estrellas primero. Esos ingenieros luego revisan manualmente el código, leen los READMEs, y deciden si quieren comunicarse.

Tu README es tu primera impresión en esa revisión. Un README que es claro, completo, y honesto hará que un reclutador piense: "Esta persona sabe cómo comunicar. Esta persona entiende que el código se escribe para humanos, no para computadoras. Quiero conocer a esta persona".

Un README que es desordenado o incompleto hace que un reclutador piense: "Esta persona no se importa por la documentación. Si este proyecto se rompe, no seré capaz de entenderlo. Pasar".

Tu README debe incluir: por qué construiste este proyecto, qué hace, cómo instalarlo, cómo usarlo, cuál es la arquitectura, cuáles son las preguntas abiertas, y cómo alguien puede contribuir. Eso es profesionalismo. Eso es cómo te presentas como ingeniero, no como estudiante que codificó por una calificación.

Y tu historial de commits importa. Si has escrito todo tu código en un commit gigante llamado "Initial Commit", eso le dice a un reclutador que no piensas como un profesional. Los verdaderos ingenieros hacen commits temprano y a menudo. Escriben mensajes de commit que cuentan la historia del proyecto: "Agregar calculadora de pase de ISS", "Arreglar análisis de elemento orbital", "Mejorar rendimiento con vectorización NumPy". Tu historial de commits es una narrativa de tu pensamiento.

Al final de esta lección, tendrás cinco proyectos, todos con excelentes READMEs, todos con historiales de commit limpios, todos con documentación que explica el dominio aeroespacial. Tendrás un perfil de GitHub que muestra a un reclutador: "Esta persona ha hecho el trabajo. Esta persona entiende el dominio. Esta persona puede lanzar código".

Eso es cuando se abren puertas. Eso es cuando consigues entrevistas en empresas que construyen el futuro. Eso es cuando dejas de ser estudiante aprendiendo a codificar, y te conviertes en ingeniero con una cartera.

Tu GitHub es tu reputación. Constrúyela como si importara. Porque importa.
